QA Bug Assistant

Project Overview

QA Bug Assistant (Jira Ticket Generator) is a QA toolkit for creating Jira-ready bug tickets in under one minute. Describe a bug by typing or voice, let AI structure the ticket, review and edit, then create the issue in Jira Cloud via MCP.

The project ships two clients — a React web dashboard and a Chrome extension (MV3) — that share the same Node API, generation logic, and Jira integration.

Web App

  • Voice dictation with Web Speech API and environment detection
  • AI ticket generation via local llama.cpp or OpenAI
  • Editable preview, title suggestions, and confidence score
  • QA context detection — browser, OS, device, environment
  • Ticket history, templates, and QA standards engine
  • One-click Jira creation from the dashboard

Chrome Extension

  • Popup on any page — captures URL, title, and timestamp
  • Optional include/exclude page title in tickets
  • Same AI generation and Jira creation as the web app
  • Draft persistence across popup closes
  • New ticket control to start a fresh report
  • Keyboard shortcut — customizable in Chrome shortcut settings
  • Auto-syncs Jira domain and project from server/.env

API Server & Jira

  • Express API on port 3001 — credentials stay in server/.env
  • MCP stdio bridge to mcp-atlassian for Jira Cloud
  • POST /api/jira/issues — create issues from web or extension
  • GET /api/config/bootstrap — extension settings sync
  • Shared payload builder and title polishing across clients

Shared Code

  • shared/jiraApi.ts — API contract (create issue, MCP test)
  • shared/generation/ — browser context, issue description
  • src/services/ticketGeneration/ — AI + rules-based generation
  • src/utils/buildJiraCreatePayload.ts — Jira create payload
  • src/utils/polishIssueTitle.ts — formal title grammar

Technology Stack

  • React 19 + TypeScript 6 + Vite 8 (web app and extension popup)
  • Tailwind CSS v4 (web app)
  • Node.js + Express 5 + Zod (API server)
  • Chrome Extension Manifest V3
  • MCP stdio → mcp-atlassian → Jira Cloud
  • AI — llama.cpp (local) or OpenAI Chat Completions
  • Vitest for unit tests

Runtime Flow

Web app:
  Dashboard → generateTicket → review → POST /api/jira/issues → MCP → Jira

Extension:
  Popup → capture tab context → generateTicket → review → POST /api/jira/issues → Jira

Shared:
  generateTicket(), buildJiraCreatePayload(), shared/jiraApi.ts
  API server :3001 (npm run api:dev) · credentials in server/.env

Extension Permissions

  • storage — Save drafts, settings, and resolved API URL
  • activeTab — Read page URL and title for bug context
  • host_permissions — localhost:3001 / 127.0.0.1:3001 for API
